Do This Before Deploying to Vercel

Learn how your app really runs before pushing it to any platform as a service

1. Try a VPS First

Spin up a cheap VPS on DigitalOcean or a similar provider. Use it for a few days to understand how hosting actually works behind the scenes.

  • Connect with ssh
  • Set up SSH keys
  • Run basic Linux commands

2. Learn Server Basics

Get comfortable with working in the terminal. Learn how to manage files, processes, and logs on a remote machine.

  • Use ls, cd, pwd, cat, grep
  • Edit files with vim, nano, or similar
  • Update your system with apt or yum

3. Run Your App Manually

Deploy your app the “hard way” at least once. Clone your repo onto the server, install dependencies, and run the server process.

  • git clone your project
  • Install dependencies (e.g. npm install)
  • Start the app (e.g. npm run start)

4. Put a Reverse Proxy in Front

Use Nginx or Caddy to serve your app over HTTPS and map a real domain name to your server.

  • Buy a domain and point DNS to your VPS IP
  • Configure Nginx/Caddy to route traffic to your app port
  • Enable HTTPS using Let’s Encrypt or Caddy’s auto TLS

5. Then Appreciate Vercel

Once you’ve done all this manually, you’ll really appreciate how platforms like Vercel, Railway, and Fly.io automate the pain away. But now you’ll understand what’s happening under the hood.

Happy coding — and don’t skip the fundamentals
